# 建模题与计算题模板

这份文档专门用于应对软件工程大题：DFD、数据字典、顺序图、状态图、N-S/PAD、McCabe 复杂度和测试覆盖。

## 一、DFD 顶层图模板

顶层 DFD 又叫系统环境图。画图时只画一个系统加工，外面放外部实体，用数据流连接。

步骤：

1. 找系统名：题干一般写“某某管理系统”“某某平台”。
2. 找外部实体：系统外部的人、组织、其他系统。
3. 找输入数据流：外部实体给系统的数据。
4. 找输出数据流：系统返回给外部实体的数据。
5. 不展开系统内部数据存储和内部加工，除非题目明确要求。

文字答题骨架：

```text
外部实体：用户、管理员、供应商……
系统：XXX 系统
数据流：
用户 -> 系统：……
系统 -> 用户：……
管理员 -> 系统：……
系统 -> 管理员：……
```

检查点：

1. 数据流名称用名词或名词短语，不用“处理”“查询”这类动词单独做数据。
2. 箭头方向要符合数据来源和去向。
3. 外部实体不能直接访问数据存储，必须通过系统或加工。

## 二、0 层 DFD 模板

0 层 DFD 是把顶层的系统加工分解为若干核心加工。

步骤：

1. 保留顶层 DFD 的外部实体。
2. 把系统分成 2 到 5 个主要加工。
3. 加入内部数据存储。
4. 画外部实体、加工、数据存储之间的数据流。
5. 检查与顶层图平衡。

常见核心加工名称：

| 业务 | 加工 |
|---|---|
| 登录 | 身份验证、用户登录处理 |
| 订单 | 请求校验、订单处理、库存更新 |
| 图书 | 借书处理、还书处理、库存查询 |
| 教学 | 注册管理、学籍管理、成绩管理、奖励管理 |
| 培训 | 信息分类、报名处理、注销处理、付款处理 |
| 购物 | 商品浏览、购物车处理、结算处理、商品管理 |

## 三、1 层 DFD 模板

1 层 DFD 是对 0 层中某个加工继续分解。

例：把“请求校验”分解为：

1. 用户信誉校验。
2. 商品库存校验。

画 1 层图时只展开被分解加工，其他加工可以作为外部接口或保持引用。要保证被分解加工的输入输出与上一层一致。

## 四、数据字典模板

基本格式：

```text
数据流名 = 数据项 + 数据项 + 数据项
重复结构 = {数据项 + 数据项}
选择结构 = [选项1 | 选项2]
```

常见例子：

```text
商品信息 = 商品ID + 商品名称 + 描述 + 价格 + 库存数量
购物车记录 = 用户ID + {商品ID + 数量}
订单信息 = 订单号 + 用户ID + 商品明细 + 订单金额 + 订单状态
图书卡片 = 分类目录号 + 流水号 + 书名 + 作者 + 价格 + 购书日期
借书信息 = [书名 | 作者] + 借书日期
```

数据存储定义模板：

```text
商品数据库：存储商品ID、名称、描述、价格、库存数量等信息，用于商品浏览和商品管理。
订单文件：存储订单号、用户信息、商品明细、金额和状态，用于订单查询、统计和报表生成。
```

## 五、顺序图模板

顺序图描述对象之间按时间顺序发送的消息。考试中可用文字说明配合图。

通用参与者：

1. 外部参与者：用户、销售员、教师、借阅人。
2. 边界对象：浏览器、终端界面、系统页面。
3. 控制对象：登录控制器、订单控制器、借阅控制器。
4. 实体/数据存储：用户信息文件、商品数据库、订单文件。

通用流程：

```text
参与者 -> 界面：提交请求
界面 -> 控制器：转发请求
控制器 -> 数据存储：查询/校验数据
数据存储 -> 控制器：返回查询结果
alt 成功
  控制器 -> 数据存储：写入/更新数据
  控制器 -> 界面：返回成功
  界面 -> 参与者：显示成功信息
else 失败
  控制器 -> 界面：返回失败原因
  界面 -> 参与者：显示失败信息
end
```

“添加商品到购物车”示例：

```text
用户 -> 在线购物系统：添加商品到购物车请求
在线购物系统 -> 商品数据库：查询商品信息并验证库存
商品数据库 -> 在线购物系统：返回商品信息/无商品
在线购物系统 -> 购物车数据库：更新购物车
购物车数据库 -> 在线购物系统：确认更新成功
在线购物系统 -> 用户：添加成功
```

## 六、状态图模板

状态图描述一个对象或系统随事件发生的状态变化。

答题步骤：

1. 找初始状态：通常是“显示首页”“主界面”“欢迎界面”。
2. 找用户触发事件：请求查询、输入 ID、确认支付等。
3. 找校验状态：校验学号、验证站点、检查库存。
4. 找成功状态：显示成绩、借阅成功、购票成功。
5. 找失败状态：显示错误信息、返回首页。
6. 找循环状态：继续查询、继续借阅、结束返回首页。

成绩查询常见状态：

```text
S0 显示首页
S1 要求学号
S2 校验学号
S3 显示学号错误信息
S4 要求课程名
S5 校验课程名
S6 显示课程名错误信息
S7 查询选课文件
S8 显示成绩
S9 询问用户操作
S10 显示查询失败
```

图书借阅常见状态：

```text
显示首页 -> 借阅图书 -> 校验图书ID -> 检查图书是否可借 -> 查询读者信息 -> 显示借阅成功 -> 是否继续借阅
```

地铁购票常见状态：

```text
显示首页 -> 购票 -> 验证起点站 -> 验证终点站 -> 确认购票信息 -> 验证支付成功 -> 显示购票成功
```

## 七、McCabe 复杂度模板

常用公式：

```text
V(G) = 判定节点数 + 1
```

如果题目给边和节点：

```text
V(G) = E - N + 2
```

答题句式：

> 该流程中有 4 个判定节点，因此 McCabe 环路复杂度 V(G) = 4 + 1 = 5。根据基本路径覆盖原则，至少需要 5 条独立路径/5 个测试用例。

基本路径列举模板：

```text
路径1：正常入口 -> 第一个校验失败 -> 返回首页
路径2：第一个校验成功 -> 第二个校验失败 -> 返回首页
路径3：前两个校验成功 -> 查询失败 -> 返回首页
路径4：查询成功 -> 用户结束 -> 返回首页
路径5：查询成功 -> 用户继续 -> 回到输入状态
```

## 八、条件组合覆盖模板

步骤：

1. 把复合判定拆成简单条件。
2. 每个简单条件取真/假。
3. n 个条件共有 `2^n` 种组合。
4. 每种组合至少一个测试用例。

例：

```text
(A >= 9) and (B < 0)
```

简单条件：

```text
C1 = A >= 9
C2 = B < 0
```

至少 4 个用例：

| 用例 | A | B | C1 | C2 | 判定结果 |
|---|---:|---:|---|---|---|
| 1 | 9 | -1 | 真 | 真 | 真 |
| 2 | 9 | 0 | 真 | 假 | 假 |
| 3 | 8 | -1 | 假 | 真 | 假 |
| 4 | 8 | 0 | 假 | 假 | 假 |

## 九、语句覆盖模板

语句覆盖只要求每条语句至少执行一次。

例：求三个整数最小值。

```text
a = 3, b = 2, c = 1
```

执行过程：

1. `min = a` 执行。
2. `b < min` 为真，`min = b` 执行。
3. `c < min` 为真，`min = c` 执行。
4. 输出 `1`。

因此 1 个测试用例即可达到语句覆盖。

## 十、N-S 图/PAD 图转换模板

给伪码：

```text
BEGIN
S1;
S2;
IF a > 5 THEN S3 ELSE S4;
n = 1;
WHILE n <= 5 DO
{
  S5;
  n = n + 1;
}
IF b > 5 THEN S6 ELSE S7;
END
```

转换思路：

1. 外层是顺序结构：S1、S2、IF、赋值、WHILE、IF。
2. 第一个 IF 是二分选择，左分支 S3，右分支 S4。
3. WHILE 是循环结构，循环体包含 S5 和 n = n + 1。
4. 第二个 IF 是二分选择，左分支 S6，右分支 S7。

如果不会画标准符号，至少用层次化文字把顺序、选择、循环表达清楚，争取步骤分。

## 十一、建模题最后检查清单

DFD：

1. 外部实体是否在系统外部。
2. 数据存储是否只能通过加工访问。
3. 数据流名称是否是数据而非动作。
4. 顶层和 0 层是否平衡。
5. 加工名称是否用动宾短语。

数据字典：

1. 是否使用 `=` 和 `+` 定义。
2. 是否列出了题干给出的字段。
3. 是否区分数据流和数据存储。

顺序图：

1. 是否有发起者。
2. 是否体现消息顺序。
3. 是否查询或更新数据文件。
4. 是否有成功/失败返回。

状态图：

1. 状态名是否描述“系统处于什么状态”。
2. 转移条件是否来自题干。
3. 成功、失败、取消、继续、结束路径是否完整。
