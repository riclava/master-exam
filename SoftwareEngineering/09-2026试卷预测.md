# 2026 年软件工程试卷预测

说明：本预测基于 `第1轮复习-全部历年真题-答案解析/软件工程` 中 2012、2013、2014、2018-2023、2025 真题回忆版及 2025 模拟卷的重复考点整理，不代表真实命题。它的用途是帮助最后阶段抓高频、练题型、减少复习盲区。

## 一、整体判断

2026 年大概率仍会围绕“基础概念辨析 + 软件设计/测试 + 建模题”命题。软件工程这部分总分通常按 30 分左右设计，近年题型可能在“简答题”和“填空/选择题”之间微调，但核心考点变化不大。

| 题型 | 预测分值 | 预测特点 |
|---|---:|---|
| 填空/选择题 | 8-10 分 | 面向对象、过程模型、需求/设计阶段任务、测试方法高频 |
| 判断题 | 5 分 | 易错概念辨析：DFD、耦合内聚、黑盒/白盒、维护、CMMI |
| 简答题 | 8-12 分 | 软件过程、测试原则、耦合/内聚、CMM/CMMI、OO 模型关系 |
| 建模/计算题 | 8-10 分 | DFD 分层、数据字典、顺序图/状态图、McCabe 复杂度 |

最可能的命题重心：结构化分析与 DFD 仍是压轴首选；面向对象与 UML 继续高频；软件测试会稳定出现；过程模型与 CMM/CMMI 多半以小题或简答出现。

## 二、2026 高概率考点排序

### 第一梯队：极可能考

1. 顶层 DFD、0 层 DFD、数据字典。
2. 概要设计与详细设计任务区分。
3. 模块内聚与耦合类型，尤其是数据耦合、控制耦合、内容耦合、过程内聚、通信内聚、顺序内聚。
4. 黑盒测试、白盒测试、边界值分析、等价类划分、测试用例选择。
5. 类与对象、封装、继承、多态、消息传递。
6. UML 关系：关联、聚合、组合、泛化、依赖。

### 第二梯队：很值得准备

1. 软件过程模型：瀑布、增量、螺旋、喷泉、敏捷、RAD、RUP。
2. CMM 五级与 CMMI 连续/阶段表示。
3. 软件生命周期阶段及各阶段任务。
4. 软件维护四类：纠错性、适应性、完善性、预防性。
5. McCabe 环路复杂度与基本路径覆盖。
6. UML 状态图、顺序图、4+1 视图模型。

### 第三梯队：低成本保分

1. 软件正确性、验证与确认、正式技术评审目标。
2. 软件配置管理、质量保证、质量控制区别。
3. 功能点、LOC、Gantt 图、风险预测。
4. E-R 模型三要素：实体、属性、联系。
5. 结构化程序设计三种基本控制结构。
6. 设计模式：单例模式、工厂模式。

## 三、2026 可能的出题方向

### 1. 建模题大概率仍考 DFD

2012、2013、2014、2018、2022、2023、2025 都出现 DFD 或数据字典，2026 年继续考的概率很高。更可能的形式是给一个业务系统，让你画顶层 DFD 和 0 层 DFD，或补充数据流、写数据字典。

必须会：

```text
外部实体：系统外的人/组织/其他系统
加工：系统内部处理功能
数据流：实体、加工、存储之间传递的数据
数据存储：系统内部文件、表、库、记录
```

答题时最关键的是平衡：顶层图中的外部输入/输出，在 0 层图中不能无故消失。

### 2. 软件设计会继续围绕“阶段 + 模块独立性”

概要设计、详细设计、内聚、耦合几乎年年出现。2026 年很可能继续用选择/判断考“哪个阶段做什么”，再用简答考“耦合定义并举例”或“高内聚低耦合”。

必须背熟：

```text
概要设计：总体结构、模块划分、模块关系、接口
详细设计：模块算法、数据结构
高内聚：模块职责单一，内部元素联系紧密
低耦合：模块间依赖少，通过清晰接口交互
```

### 3. 面向对象可能考“基础概念 + UML 关系”

2025 回忆版已经考了类/对象、消息传递、4+1 视图、封装、继承、最小界面。2026 可能继续考 OO 基础，但会换成 UML 关系或状态/顺序图。

重点组合：

```text
类是对象的抽象；对象是类的实例
对象之间通过消息传递交互
封装实现信息隐蔽
继承支持复用
多态是不同对象对同一消息不同响应
泛化 = 继承
聚合/组合 = 整体-部分
```

### 4. 软件测试仍会考易错概念

软件测试的题目通常不难，但非常爱考反向判断：

```text
测试不能证明软件无错
测试用例不能只选合理输入
边界值分析是黑盒测试
单元测试主要测模块内部逻辑
集成测试主要测模块接口
McCabe 复杂度 = 判定节点数 + 1
```

如果出计算题，大概率是 McCabe 环路复杂度、条件组合覆盖或语句覆盖。

### 5. 过程模型可能结合近年敏捷/RUP/CMMI

2025 回忆版考敏捷，2020/2021 考 RUP、RAD、CMMI，2023 考 CMM。2026 可能不会只考瀑布这种基础点，而是把“敏捷、螺旋、喷泉、CMMI”放进填空或选择。

重点辨析：

```text
瀑布：需求明确稳定
螺旋：风险分析 + 原型
喷泉：面向对象 + 迭代 + 无间隙
敏捷：迭代增量 + 反馈 + 减少不必要文档
RUP：角色回答“谁做”
CMMI：既支持阶段式，也支持连续式
```

## 四、2026 预测卷

### 一、填空题（每空 1 分，共 5 分）

1. 类是对象的____，对象是类的____。

2. 结构化分析模型通常由数据流图、____和加工说明组成。

3. CMM 的第三级是____级。

4. 面向对象 4+1 视图模型中的“+1”通常指____视图。

预测答案：抽象；实例；数据字典；已定义；场景/用例。

### 二、单项选择题（每题 1 分，共 5 分）

1. 软件详细设计阶段的主要任务是（ ）

A. 确定系统总体结构  
B. 确定模块间接口  
C. 确定模块算法和数据结构  
D. 编写需求规格说明书

预测答案：C。

2. 模块间通过参数传递基本类型数据，这种耦合属于（ ）

A. 数据耦合  
B. 控制耦合  
C. 公共耦合  
D. 内容耦合

预测答案：A。

3. 下列测试方法中属于黑盒测试的是（ ）

A. 语句覆盖  
B. 基本路径覆盖  
C. 边界值分析  
D. 判定覆盖

预测答案：C。

4. UML 中表示一般化/特殊化关系的是（ ）

A. 依赖  
B. 泛化  
C. 聚合  
D. 实现

预测答案：B。

5. 螺旋模型相对于瀑布模型和演化模型增加的核心活动是（ ）

A. 风险分析  
B. 代码复用  
C. 数据建模  
D. 配置审计

预测答案：A。

### 三、判断题（每题 1 分，共 5 分）

1. DFD 主要用于描述系统的控制流程。（ ）

预测答案：错。DFD 描述数据流动，不描述控制流程。

2. 模块的内聚性越高越好，耦合性越低越好。（ ）

预测答案：对。

3. 测试没有发现错误，可以证明软件没有错误。（ ）

预测答案：错。测试只能说明当前测试范围内未发现错误。

4. 软件维护包括纠错性维护、适应性维护、完善性维护和预防性维护。（ ）

预测答案：对。

5. CMMI 只支持阶段式过程改进，不支持连续式过程改进。（ ）

预测答案：错。CMMI 同时支持阶段式和连续式表示。

### 四、简答题（每题 4 分，共 12 分）

#### 1. 简述概要设计和详细设计的区别。

预测要点：

概要设计确定软件系统总体结构、模块划分、模块间关系和接口，关注系统整体架构，输出概要设计说明书或优化的软件构造图。

详细设计在概要设计基础上，对每个模块内部进行设计，确定算法、数据结构、处理流程和接口细节，输出详细设计说明书、流程图、N-S 图、PAD 图或伪码。

#### 2. 简述黑盒测试与白盒测试的区别，并各举两种常见方法。

预测要点：

黑盒测试不关心程序内部结构，依据功能规格说明检查输入输出和功能是否正确。常见方法有等价类划分、边界值分析、因果图、判定表。

白盒测试依据程序内部逻辑结构、控制流和代码路径设计测试用例。常见方法有语句覆盖、判定覆盖、条件覆盖、基本路径覆盖。

#### 3. 给出模块耦合的定义，并举例说明 3 种耦合。

预测要点：

模块耦合是模块之间相互依赖或关联的程度。耦合越低，模块独立性越强，可维护性和可复用性越好。

可举三种：

1. 数据耦合：模块间通过参数传递基本数据项。
2. 控制耦合：模块间传递控制标志，影响被调模块执行逻辑。
3. 公共耦合：多个模块共享公共数据区或全局变量。
4. 内容耦合：一个模块直接访问或修改另一个模块内部数据，是最差耦合。

### 五、建模与计算题（共 8 分）

问题陈述：某在线预约管理系统包含以下功能。

用户可以提交预约请求，预约请求包括用户编号、预约项目、预约时间。系统收到请求后，先查询“用户信息表”检查用户状态是否正常，再查询“预约资源表”检查该时间段资源是否可用。若校验通过，系统生成预约记录并写入“预约记录文件”，同时向用户返回预约成功信息。若资源不可用，系统向用户返回预约失败原因。

管理员可以维护预约资源信息，包括新增、修改和删除资源可用时间段。部门经理可以查询“预约记录文件”，统计预约情况并生成预约统计报表。

#### 1. 用结构化分析方法给出该系统的顶层 DFD。（3 分）

预测要点：

外部实体：

1. 用户。
2. 管理员。
3. 部门经理。

系统：

在线预约管理系统。

数据流：

```text
用户 -> 系统：预约请求
系统 -> 用户：预约成功信息/预约失败原因
管理员 -> 系统：资源维护信息
系统 -> 管理员：维护结果
部门经理 -> 系统：预约统计查询请求
系统 -> 部门经理：预约统计报表
```

#### 2. 给出 0 层 DFD 的核心加工和数据存储。（3 分）

预测要点：

核心加工可分为：

1. 预约请求校验。
2. 预约记录处理。
3. 资源信息维护。
4. 预约统计报表生成。

数据存储：

1. 用户信息表。
2. 预约资源表。
3. 预约记录文件。

关键数据流：

```text
预约请求 -> 预约请求校验
预约请求校验 -> 用户信息表：查询用户状态
预约请求校验 -> 预约资源表：查询资源可用情况
校验通过 -> 预约记录处理
预约记录处理 -> 预约记录文件：写入预约记录
预约统计报表生成 -> 预约记录文件：查询预约记录
```

#### 3. 若根据该预约流程画程序控制流图，其中有 3 个判定：用户状态是否正常、资源是否可用、是否继续预约。求 McCabe 环路复杂度。（2 分）

预测解法：

判定节点数为 3。

```text
V(G) = 判定节点数 + 1 = 3 + 1 = 4
```

因此 McCabe 环路复杂度为 4，基本路径覆盖至少需要 4 条独立路径。

## 五、备选高概率简答题

### 1. 软件生命周期阶段及任务

按“可行性研究与计划、需求分析、设计、编码、测试、维护、退役”回答即可。设计阶段要区分概要设计与详细设计，维护要写四类维护。

### 2. 面向对象三大模型关系

对象模型描述静态结构，是基础；动态模型描述对象状态变化和消息交互；功能模型描述系统功能目标。三者从静态、动态、功能三个角度共同描述系统。

### 3. CMM 五级模型

初始级、可重复级、已定义级、已管理级、优化级。可展开某一级关键过程域，如可重复级包括需求管理、项目计划、项目跟踪与监控、质量保证、配置管理等。

### 4. 软件测试原则

尽早测试、全面测试、重点测试、独立测试、及时反馈、回归测试、制定测试计划、保留测试记录。

### 5. 设计模式

设计模式是对反复出现的软件设计问题总结出的通用解决方案。单例模式保证一个类只有一个实例；工厂模式封装对象创建过程，降低对象创建和使用之间的耦合。
