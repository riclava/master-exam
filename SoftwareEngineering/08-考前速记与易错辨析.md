# 考前速记与易错辨析

这份文档适合考前最后一遍快速扫。每条都是历年题里容易混淆的点。

## 一、阶段任务速记

需求分析：定义需求，建立系统逻辑模型，输出需求规格说明书。

概要设计：确定总体结构、模块划分、模块关系和接口，输出优化的软件构造图。

详细设计：确定模块算法和数据结构，常用流程图、N-S 图、PAD 图、伪码。

编码：把详细设计转换为代码。

单元测试：测模块内部逻辑。

集成测试：测模块接口和协作，主要依据概要设计说明书。

系统测试：测整个系统是否符合需求规格说明。

验收测试：用户确认软件是否满足实际需求。

维护：交付后纠错、适应、完善、预防。

## 二、过程模型速记

瀑布模型：需求明确稳定，线性顺序，阶段分明，文档驱动。

增量模型：需求可分组，分批交付，每个增量可按瀑布式开发。

演化模型：小步迭代，逐步逼近最终系统。

螺旋模型：瀑布 + 演化 + 风险分析，基于原型，高风险项目常用。

喷泉模型：面向对象，迭代，无间隙。

敏捷模型：迭代增量，减少不必要文档，分析/设计/编码/测试结合，持续反馈。

RAD：快速应用开发，业务建模、数据建模、流程建模。

RUP：谁做是角色，怎么做是工作流，做什么会产生制品。

## 三、CMM/CMMI 速记

CMM 五级：

```text
初始级 -> 可重复级 -> 已定义级 -> 已管理级 -> 优化级
```

第三级是已定义级。

CMM 支撑软件质量三要素：人员素质、过程、技术。

CMMI 既支持阶段式表示，也支持连续式表示。

连续式能力等级：

```text
0 不完整级
1 已执行级
2 已管理级
3 已定义级
4 量化管理级
5 最优化级
```

## 四、DFD 速记

DFD 四要素：

```text
外部实体、加工、数据流、数据存储
```

DFD 描述数据流，不描述控制流程。

顶层 DFD 又叫系统环境图，表示系统边界和输入输出。

结构化分析模型不只是 DFD，还包括数据字典和加工说明。

数据字典写法：

```text
数据流 = 数据项 + 数据项
重复 = {数据项 + 数据项}
选择 = [选项1 | 选项2]
```

## 五、内聚速记

由低到高：

```text
偶然/巧合 < 逻辑 < 时间 < 过程 < 通信 < 顺序 < 功能
```

题眼：

偶然：没什么联系。

逻辑：逻辑相关，靠控制参数选择一个功能。

过程：必须按特定次序执行。

通信：操作同一数据区或同一输入/输出。

顺序：前一处理输出是后一处理输入。

功能：共同完成一个单一功能。

## 六、耦合速记

由低到高：

```text
数据 < 标记 < 控制 < 外部 < 公共 < 内容
```

题眼：

数据耦合：传基本数据参数，最好。

标记耦合：传完整数据结构，只用一部分。

控制耦合：传 flag/mode 控制被调模块。

公共耦合：共享全局数据。

内容耦合：直接访问或修改别的模块内部数据，最差。

设计目标：高内聚、低耦合。

## 七、OO 和 UML 速记

类是对象的抽象，对象是类的实例。

对象 = 数据 + 操作，体现封装。

对象之间通过消息传递交互。

不同对象对同一消息不同响应，是多态。

子类复用父类属性和方法，是继承。

UML 关系：

```text
关联：长期语义联系
聚合：弱整体-部分，部分可独立存在
组合：强整体-部分，部分生命周期依赖整体
泛化：继承，一般/特殊
依赖：临时使用
实现：类实现接口
```

学校-教师、剧组-演员：聚合。

整体/部分不能用依赖表示。

4+1 视图：

```text
逻辑视图、物理视图、开发视图、进程视图、场景视图
```

## 八、测试速记

测试目的：发现错误，不是证明无错。

黑盒测试：依据功能规格说明，关注功能。

白盒测试：依据内部逻辑、控制流、路径。

等价类和边界值：黑盒测试。

边界值不是白盒。

测试用例不能只选合理输入，还要选不合理输入、边界、异常。

条件组合覆盖：n 个条件至少 `2^n` 种组合。

McCabe：

```text
V(G) = 判定节点数 + 1
V(G) = E - N + 2
```

基本路径覆盖所需独立路径数通常等于 McCabe 复杂度。

## 九、维护速记

维护四类：

1. 纠错性维护：修错误。
2. 适应性维护：适应环境变化。
3. 完善性维护：增强功能或性能。
4. 预防性维护：提高未来可维护性和可靠性。

“维护不包括纠错”是错的。

## 十、质量和管理速记

正式技术评审的目标是发现错误，不是评价程序员效率。

验证：是否正确地做了产品。

确认：是否做了正确的产品。

软件配置管理是保护伞活动，贯穿生命周期，用于控制变更。

质量控制不是软件配置管理活动。

Gantt 图：进度控制。

功能点分析法：规模估算。

LOC 和功能点：最常用规模度量。

风险预测：可能性 + 影响程度，不是风险消除方法。

## 十一、最常见判断题反向记忆

“结构化分析模型就是 DFD”是错的。

“DFD 描述控制流程”是错的。

“边界值分析是白盒测试”是错的。

“测试没发现问题证明无错”是错的。

“测试用例只需合理输入”是错的。

“数据耦合是最差耦合”是错的。

“内容耦合是好耦合”是错的。

“耦合越高越好”是错的。

“类是对象的实例”是错的。

“接收消息的对象必须回答发送者”是错的。

“CMMI 不支持连续式改进”是错的。

“单元测试主要发现模块接口错误”是错的。

“维护不包括纠错”是错的。
