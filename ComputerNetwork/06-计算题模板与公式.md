# 计算题模板与公式

这份文档只放历年反复出现的计算套路。考试时先识别题型，再套模板。

## 一、时延与利用率

### 1. 基本公式

`发送时延 = 数据长度(bit) / 发送速率(bit/s)`

`传播时延 = 链路长度 / 信号传播速度`

`RTT = 2 × 单程传播时延`，但题目直接给 RTT 时直接用题目值。

`时延带宽积 = 带宽 × 传播时延`

### 2. 停止等待利用率

停止等待一次只发一帧，等 ACK 后再发下一帧，近似：

`U = 发送时延 / (发送时延 + RTT)`

如果要提高利用率，答案通常是“流水线技术”或“滑动窗口协议”。

### 3. 反推帧长

题目给发送时延和速率：

`帧长(bit) = 发送时延 × 速率`

再除以 8 得字节数。

例：100Mbps，发送时延 2ms：

`100 × 10^6 × 2 × 10^-3 = 200000 bit = 25000 Byte`

## 二、CSMA/CD

### 1. 最小帧长

`最小帧长(bit) = 数据率(bit/s) × 争用期(s)`

`争用期 = 2 × 最远端单程传播时延`

如果题目给的是链路长度和传播速度：

`单程传播时延 = 距离 / 传播速度`

### 2. 二进制指数退避

第 k 次冲突后随机等待 r 个争用期：

`r ∈ [0, 2^k - 1]`

以太网实际采用截断二进制指数退避，k 增大到一定值后不再继续扩大。

## 三、滑动窗口序号位

### 1. Go-Back-N

序号字段 n bit，序号空间 `2^n`。

`发送窗口最大 W = 2^n - 1`

反过来，如果已知窗口 W，至少需要 n 满足：

`2^n >= W + 1`

### 2. Selective Repeat

为避免新旧帧混淆：

`发送窗口 = 接收窗口 <= 2^(n-1)`

反过来，若窗口至少 W，至少需要 n 满足：

`2^(n-1) >= W`

### 2. 网络地址与广播地址

网络地址：`IP & 掩码`

广播地址：网络位不变，主机位全 1。

可用地址范围：网络地址 + 1 到广播地址 - 1。

### 3. 子网划分

需要 N 个子网：找 `s` 使 `2^s >= N`。

每子网至少 M 台主机：找 `h` 使 `2^h - 2 >= M`。

新前缀 = 原前缀 + s，同时剩余主机位要不少于 h。

### 4. 路由汇聚

将多个网络地址转二进制，找最长共同前缀。共同前缀后补 0，得到汇聚地址。

检查：汇聚地址块必须覆盖全部给定网络，且前缀越长越精确。

## 五、最长前缀匹配

模板：

1. 目的 IP 与每条路由掩码按位与。
2. 等于该条目的网络地址则匹配。
3. 多条匹配取前缀最长。
4. 无匹配走默认路由。

答题时写“按最长前缀匹配原则选择下一跳”。

## 六、IP 分片

### 1. 单次分片

已知 IP 数据报总长度 T，首部 H，MTU M：

`数据部分 D = T - H`

`每片最大数据 = M - H`

除最后一片外，每片数据长度向下取 8 的倍数。

片数：

`ceil(D / 每片最大数据)`

每片总长度：

`本片数据长度 + H`

片偏移：

`本片数据起始位置 / 8`

MF 标志：最后一片为 0，其余为 1。

### 2. 经过多个 MTU 网络

先按第一个 MTU 分片；进入更小 MTU 的网络后，每个已有分片还可能继续分片。重组只在最终目的主机发生。

## 七、RIP/距离向量

收到邻居 C 的距离向量，且本路由器到 C 的代价为 d：

`经 C 到目标网络的距离 = C 通告距离 + d`

更新规则：

1. 原表没有该目标：加入，下一跳为 C。
2. 原下一跳就是 C：无论距离变大变小，都更新。
3. 原下一跳不是 C：只有新距离更小才替换。
4. 距离达到 16：不可达。

## 八、TCP 序号、确认号

### 1. 数据长度

相邻两个 TCP 段序号为 `seq1` 和 `seq2`：

`第一个段数据长度 = seq2 - seq1`

### 2. 确认号

收到从 `seq` 开始、长度为 `len` 的连续数据：

`ACK = seq + len`

若中间有丢失，ACK 停在第一个缺失字节。

## 九、TCP 拥塞控制

### 1. 慢开始

按 RTT 看，cwnd 近似翻倍：

`1, 2, 4, 8, 16...`

累计发送：

`前 n 轮 = 2^n - 1`

### 2. 拥塞避免

每 RTT 线性增加 1 MSS：

`W, W+1, W+2...`

### 3. 超时

`ssthresh = 当前 cwnd / 2`

`cwnd = 1 MSS`

重新慢开始。

### 4. 三个重复 ACK

触发快重传、快恢复：

`ssthresh = 当前 cwnd / 2`

通常 cwnd 降到门限附近，然后进入拥塞避免。若题目给图，以图为准。

## 十、传输层协议最大数据率

题型：序号 n 位，最大报文段长度 MSS 字节，最大生存期 MSL 秒。

`最大数据率 = 2^n × MSS × 8 / MSL`

例：n=4，MSS=512B，MSL=60s：

`16 × 512 × 8 / 60 = 1092.27 bps`

## 十一、文件传输时间

常见组成：

`总时间 = 握手/建立连接时延 + 数据发送时间 + 传播时延 + 可能的 RTT 等待`

数据发送时间：

`文件大小(bit) / 链路速率(bit/s)`

若题目说每个 RTT 只能发送 k 个分组：

1. 算总分组数 `N = ceil(文件大小 / 分组大小)`。
2. 算需要多少批 `ceil(N / k)`。
3. 每批通常消耗一个 RTT，再加题目给定的握手和最后传播时延。

具体以题目叙述为准。
