# 传输层

传输层是第二个核心主战场。历年高频点集中在 TCP 序号/确认号、端口与套接字、UDP 特点、TCP 拥塞控制和传输层最大数据率。

## 一、端口、复用与套接字

传输层提供进程到进程的通信。IP 地址标识主机，端口号标识主机上的应用进程。

复用：发送方多个应用进程的数据都可交给传输层，由传输层加上端口号后交给网络层。

分用：接收方传输层根据目的端口号把数据交给正确应用进程。

套接字 Socket：IP 地址 + 端口号。TCP 连接的两个端点就是两个套接字。

## 二、TCP 与 UDP 对比

| 项目 | TCP | UDP |
|---|---|---|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠交付 | 尽最大努力交付 |
| 数据形式 | 字节流 | 报文 |
| 控制机制 | 序号、确认、重传、流量控制、拥塞控制 | 基本无可靠/拥塞控制 |
| 首部 | 较复杂 | 简单 |
| 适用 | 文件传输、Web、邮件等可靠场景 | DNS、实时音视频、简单请求响应 |

UDP 特点问答题可写：

1. 无连接，发送前不建立连接。
2. 尽最大努力交付，不保证可靠、不保证顺序、不自动重传。
3. 面向报文，保留应用层报文边界。
4. 首部开销小，时延低。
5. 支持一对一、一对多、多对一、多对多通信。
6. 适合 DNS、SNMP、实时音视频、广播/组播等。

## 三、TCP 可靠传输

TCP 提供端到端可靠传输，核心机制：

序号：TCP 把数据看成连续字节流，序号字段表示本报文段数据部分第一个字节的序号。

确认号：表示接收方期望收到的下一个字节序号。若已连续收到 1 到 1000 字节，则确认号为 1001。

累计确认：接收方只确认按序连续收到的数据。若前面的段丢失，后面的段到了，也仍确认丢失段的起始序号。

超时重传：超时未收到 ACK，重传相关报文段。

校验和：检测传输差错。

流量控制：接收方通过接收窗口 rwnd 告知还能接收多少数据，避免缓冲区溢出。

拥塞控制：发送方根据网络拥塞状况调整拥塞窗口 cwnd。

发送窗口：

`发送窗口 = min(接收窗口 rwnd, 拥塞窗口 cwnd)`

## 四、TCP 序号与 ACK 计算

### 1. 连续报文段

如果第一个 TCP 报文段序号为 750，第二个相邻报文段序号为 1100，则第一个报文段数据长度为：

`1100 - 750 = 350 字节`

如果第一个报文段序号为 200，第二个为 1100，则第一个数据长度为 900 字节。

### 2. ACK 号

如果 A 发送 `seq=300`，有效载荷 100 字节，则 B 正确收到后确认号：

`ACK = 300 + 100 = 400`

如果第一个报文段丢失，第二个报文段先到，接收方仍期望第一个报文段的起始字节，确认号仍为第一个丢失段的序号。

常见陷阱：ACK 不是“收到的最后一个序号”，而是“下一个期望序号”。

## 五、流量控制

流量控制是为了避免接收方来不及接收导致缓冲区溢出。TCP 通过接收窗口实现：接收方在 ACK 中通告窗口大小，发送方不能超过该窗口发送未确认数据。

注意区分：

流量控制：保护接收方，关注接收缓存。

拥塞控制：保护网络，关注路由器队列、链路负载和丢包。

## 六、拥塞控制

必须会写四个算法：慢开始、拥塞避免、快重传、快恢复。

### 1. 慢开始

连接开始或超时后，cwnd 从 1 MSS 开始。每收到 ACK 增加 1 MSS；按 RTT 看，cwnd 近似翻倍：

`1, 2, 4, 8, 16 ...`

直到达到慢开始门限 `ssthresh`，转入拥塞避免。

### 2. 拥塞避免

cwnd 线性增长，通常每 RTT 增加 1 MSS：

`16, 17, 18, 19 ...`

### 3. 超时

超时表示较严重拥塞：

`ssthresh = 当前 cwnd / 2`

`cwnd = 1 MSS`

然后重新慢开始。

### 4. 三个重复 ACK

收到 3 个重复 ACK，触发快重传，不等超时立即重传丢失段。

Reno 常见处理：`ssthresh = 当前 cwnd / 2`，进入快恢复/拥塞避免；有些教材题会把 `cwnd` 设为门限附近。考试按题目图或解析口径作答，但核心是“阈值减半，不像超时那样直接从 1 慢开始”。

### 5. 累计发送段数

慢开始阶段每轮发送量为 `1,2,4,8...`，前 n 轮累计：

`1 + 2 + 4 + ... + 2^(n-1) = 2^n - 1`

判断某个报文段在哪一轮发送时，用累计范围：

第 1 轮发 1；
第 2 轮发 2-3；
第 3 轮发 4-7；
第 4 轮发 8-15；
第 5 轮发 16-31；
第 6 轮发 32-63；
第 7 轮发 64-127。

所以报文段 80 在第 7 轮。

## 七、传输层最大数据率

题型：设计传输层协议，序号字段 n 位，最大报文段长度 MSS，段最大生存期 MSL，求最大数据率。

思想：在 MSL 时间内不能让序号空间用完后又被旧段混淆。

公式：

`最大数据率 = 2^n × MSS × 8 / MSL`

例：序号 4 位，MSS = 512 字节，MSL = 60s：

`2^4 × 512 × 8 / 60 = 1092.27 bps`

## 八、常见失分点

1. TCP 是面向字节流，不是面向报文。
2. UDP 端口号用于区分应用进程，不是标识主机。
3. TCP 发送窗口取接收窗口和拥塞窗口较小值。
4. ACK 号是期望下一个字节，不是已收到字节数。
5. 超时和 3 个重复 ACK 的窗口变化不同。
6. 慢开始不是“增长很慢”，而是从小窗口开始指数增长。
