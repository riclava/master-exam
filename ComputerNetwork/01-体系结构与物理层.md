# 体系结构与物理层

## 一、网络体系结构必须掌握

### 1. 协议三要素

协议是通信双方共同遵守的规则，三要素是语法、语义、同步。

语法：规定数据与控制信息的结构、格式、字段顺序、编码方式。例如 TCP 首部有哪些字段、字段长度是多少。

语义：规定字段和控制信息的含义，以及通信双方要执行的动作。例如 TCP 确认号表示“期望收到的下一个字节序号”。

同步：规定事件发生的先后顺序。例如 TCP 三次握手必须按 SYN、SYN+ACK、ACK 的顺序进行。

常见陷阱：三要素缺一不可，不能说语义和同步比语法更重要。

### 2. OSI 层次与 PDU

| 层次 | 主要职责 | PDU/数据单位 | 常考点 |
|---|---|---|---|
| 物理层 | 比特传输、编码、速率、介质 | 比特流 | 带宽、发送时延、传播时延、奈奎斯特 |
| 数据链路层 | 成帧、差错检测、介质访问、相邻节点传输 | 帧 | 以太网、交换机、VLAN、CSMA、滑动窗口 |
| 网络层 | 路由选择、逻辑寻址、网络互联、分片 | 分组/IP 数据报 | IP、CIDR、路由协议、分片 |
| 传输层 | 端到端进程通信、可靠传输、流量/拥塞控制 | TCP 报文段、UDP 用户数据报 | TCP/UDP、端口、窗口 |
| 应用层 | 面向应用的网络服务 | 应用报文 | DNS、HTTP、FTP、SMTP |

服务访问点 SAP：上层访问下层服务的入口。不同层的 SAP 有不同标识，传输层常用端口号标识应用进程。

封装效率：如果应用层数据为 `D`，各层首部总开销为 `H`，总传输字节数为 `D + H`，传输效率为 `D / (D + H)`。

## 二、物理层核心概念

### 1. 发送时延、传播时延、RTT

发送时延也叫传输时延，是把整个帧推入链路所需时间：

`发送时延 = 数据长度(bit) / 发送速率(bit/s)`

传播时延是信号在链路上传播所需时间：

`传播时延 = 链路长度 / 信号传播速度`

往返时延 RTT 通常约等于两倍单程传播时延，但题目若直接给 RTT，应直接使用题目值。

常见单位换算：

| 单位 | 换算 |
|---|---|
| 1 Byte | 8 bit |
| 1 KB | 可能按 1000B 或 1024B，按题目语境；真题常用近似 |
| 1 Mbps | `10^6 bit/s` |
| 1 ms | `10^-3 s` |
| 1 μs | `10^-6 s` |

### 2. 带宽、时延带宽积

时延带宽积表示“链路管道中最多能容纳多少比特”：

`时延带宽积 = 带宽 × 单程传播时延`

如果题目问“为了让管道满载，窗口至少多大”，通常要算：

`窗口容量(bit) >= 带宽 × RTT`

再除以每帧大小得到至少多少帧。

### 3. 奈奎斯特定理

无噪声信道最大数据率：

`Rmax = 2B log2(V)`

其中 `B` 是带宽，`V` 是码元可取的离散电平数。2012 年考过类似“带宽 + 调制电平数”。

必须会判断：题目说无噪声，用奈奎斯特；题目给信噪比并问极限速率，才考虑香农公式。

### 4. 编码

以太网发送数据常考曼彻斯特编码：每个比特中间有跳变，便于同步，但编码效率不如简单二进制编码。

### 5. 最小帧长与传播时延的关系

CSMA/CD 中，发送端必须在发送完最短帧之前还能检测到最远端冲突，所以：

`最小帧长(bit) = 数据率(bit/s) × 争用期(s)`

`争用期 = 2 × 最远端单程传播时延`

这部分虽然归在数据链路层，但计算用的是物理层的时延公式。

## 三、常见失分点

1. 把发送时延和传播时延混为一谈。发送时延看数据长度和速率，传播时延看距离和传播速度。
2. RTT 和单程传播时延用错。停止等待、窗口满载通常用 RTT；CSMA/CD 最小帧长用往返传播时延。
3. 字节和比特忘记乘 8。
4. `Mbps` 误写成 `MB/s`。
5. 奈奎斯特公式中 `log2(V)` 的 `V` 是码元种类数，不是带宽。

## 四、你要能完成的题

给定链路长度、传播速度、帧长、速率，能算发送时延、传播时延、RTT、链路利用率。

给定发送时延和速率，能反推数据帧长度。

给定无噪声带宽和调制电平数，能用奈奎斯特公式算最大数据率。

给定 CSMA/CD 网络长度、速率、传播速度，能算最小帧长。
