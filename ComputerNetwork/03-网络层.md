# 网络层

网络层是历年最核心的得分区。你必须把 IP 地址计算、CIDR、路由协议、IP 分片练到条件反射。

## 一、IP 地址与子网

### 1. 地址分类

传统分类地址按第一个字节判断：

| 类别 | 首字节范围 | 默认掩码 |
|---|---:|---|
| A 类 | 1-126 | 255.0.0.0 |
| B 类 | 128-191 | 255.255.0.0 |
| C 类 | 192-223 | 255.255.255.0 |

真题常考“某地址属于哪类”。例如 119.x.x.x 属于 A 类。

### 2. 子网掩码与前缀长度

前缀长度就是子网掩码中连续 1 的个数。例如：

| 掩码 | 前缀 |
|---|---:|
| 255.255.255.0 | /24 |
| 255.255.255.192 | /26 |
| 255.255.255.224 | /27 |
| 255.255.252.0 | /22 |
| 255.255.248.0 | /21 |
| 255.255.240.0 | /20 |
| 255.255.224.0 | /19 |
| 255.255.192.0 | /18 |
| 255.248.0.0 | /13 |

### 3. 网络地址、广播地址、可用地址

网络地址：IP 与子网掩码按位与。

广播地址：网络位不变，主机位全置 1。

可用主机地址：网络地址 + 1 到广播地址 - 1。

可用主机数：

`2^主机位数 - 2`

子网块大小：如果某字节掩码不是 255 或 0，块大小为 `256 - 掩码值`。例如 255.255.224.0 的第三字节块大小为 32，范围是 0-31、32-63、64-95、96-127 等。

### 4. 子网划分

给定“需要划分 N 个子网，每个子网至少 M 台主机”：

1. 找子网位 s：`2^s >= N`。
2. 找主机位 h：`2^h - 2 >= M`。
3. 检查原始地址块位数是否够用。
4. 新前缀 = 原前缀 + s；同时也应满足剩余主机位 h。

例：`192.168.1.0/24` 划分 4 个子网，每个至少 30 台主机。4 个子网需要 2 位，30 台主机需要 5 位，前缀为 /26，掩码 `255.255.255.192`。

### 5. 多个 C 类网络数量

一个默认 C 类网络主机位 8 位，可用主机数 `2^8 - 2 = 254`。若要支持 800 台主机，至少需要 `ceil(800/254)=4` 个 C 类网络。

## 二、CIDR 与路由汇聚

CIDR 是无分类域间路由，写作 `IP地址/前缀长度`。前缀长度越长，网络范围越小；前缀越短，网络范围越大。

### 1. CIDR 地址范围

`/20` 表示主机位 12 位，地址总数 `2^12 = 4096`。如果是 `192.168.10.0/20`，真正覆盖范围要按第三字节的块边界判断。`/20` 第三字节块大小为 16，所以 10 落在 0-15 块，范围是 `192.168.0.0` 到 `192.168.15.255`。

常见陷阱：CIDR 地址块的起始地址必须是块边界。题目给的 `192.168.10.0/20` 可能不是规范网络地址，但判断范围时要按前缀归属块算。

### 2. 路由汇聚步骤

1. 把多个网络地址写成二进制。
2. 找从左到右完全相同的最长前缀。
3. 前缀后面全部置 0，得到汇聚网络地址。
4. 前缀长度就是共同前缀位数。

例：`10.18.96.0/21`、`10.18.104.0/21`、`10.18.112.0/21`、`10.18.120.0/21` 的共同前缀为 /19，汇聚为 `10.18.96.0/19`。

## 三、最长前缀匹配

路由器转发 IP 分组时，根据目的 IP 查路由表。若多条路由匹配，选择前缀最长的一条；若没有匹配，走默认路由。

解题步骤：

1. 对目的 IP 和每条路由掩码做按位与。
2. 结果等于该条目的目的网络地址，则匹配。
3. 多条匹配取前缀最长。
4. 都不匹配则走 `Default`。

路由器根据 IP 地址和路由表转发分组，不根据 MAC 地址转发分组；MAC 地址转发是交换机的职责。

## 四、IP 分片与 MTU

MTU 是链路层帧能承载的最大 IP 数据报长度，通常包括 IP 首部。以太网常见 MTU 为 1500 字节。

IPv4 分片规则：

1. 每个分片都有自己的 IP 首部。
2. 分片总长度不超过下一跳链路 MTU。
3. 除最后一片外，每片数据部分长度必须是 8 字节的整数倍。
4. 片偏移字段单位是 8 字节。
5. 分片只在目的主机重组，中间路由器不重组。

计算模板：

`每片最大数据 = MTU - IP首部长度`

如果不是最后一片，要向下取 8 的倍数。

例：MTU 700，IP 首部 20，则每片数据最大 680，680 正好是 8 的倍数。若数据部分 2380 字节，则分为 680、680、680、340 四片。

IPv6 特点：

1. 地址长度 128 位，冒号十六进制表示。
2. 基本首部固定，取消 IPv4 选项字段，改用扩展首部。
3. 路由器不分片，只能源主机分片。
4. 支持流标号。
5. 本地链路单播地址只能在本地链路使用，不能在互联网通信。

## 五、路由协议

### 1. IGP 与 EGP

IGP：自治系统内部路由协议，如 RIP、OSPF。

EGP：自治系统之间路由协议，当前常用 BGP。

自治系统 AS：由一个组织或运营者管理、采用统一路由策略的一组网络和路由器，拥有唯一 AS 编号。

### 2. RIP

RIP 是内部网关协议，基于距离向量算法。

核心特征：

1. 度量是跳数。
2. 最大有效跳数为 15，16 表示不可达。
3. 周期性与邻居交换路由表。
4. 更新规则：经邻居到某网络的距离 = 邻居通告距离 + 到该邻居的代价。

RIP 距离向量计算题步骤：

1. 对邻居发来的每个目的网络，距离先加上到邻居的链路代价。
2. 若原表无该目的网络，添加。
3. 若下一跳原本就是该邻居，更新为邻居通告的新距离。
4. 若下一跳不是该邻居，只有新距离更小时才替换。
5. 原表中邻居未通告的目的网络保持不变。

### 3. OSPF

OSPF 是内部网关协议，基于链路状态算法。

必须掌握：

1. 通过泛洪 LSA 同步链路状态信息。
2. 每台路由器形成链路状态数据库 LSDB。
3. 用 Dijkstra 最短路径算法计算路由。
4. 支持分层区域。
5. 连接多个区域的路由器叫 ABR，区域边界路由器。
6. OSPF 不是距离向量协议，不是 EGP。

### 4. BGP

BGP 是外部网关协议，用于不同自治系统之间。

必须掌握：

1. BGP 使用路径向量算法，不是距离向量算法。
2. BGP 交换的是可达性和 AS 路径等信息。
3. BGP 不总是选择最短路径，而是受策略影响。
4. BGP 支持 CIDR。
5. BGP 交换路由信息的节点数量级与自治系统数量相关，而不是所有路由器数量。

## 六、ARP、NAT、网络互联

ARP 用于根据 IP 地址解析同一链路上的 MAC 地址。ARP 请求是广播，ARP 响应通常是单播。注意：ARP 不是 UDP 协议，属于网络接口层/链路相关协议；若题目把重点放在“广播报文过多”，应抓住 ARP 请求广播这个考点。

NAT 是网络地址转换，通常在路由器上把私有地址转换为公有地址，使多台内网主机共享公网地址访问互联网。NAT 解决地址不足，但不等同于加密或安全隧道。

路由器功能：连接不同网络，根据路由表和最长前缀匹配转发 IP 分组，可实现 NAT；不只是连接两个网络。

## 七、常见失分点

1. 子网地址要按位与，不是直接看 IP 前几段。
2. 广播地址是主机位全 1，不是网络地址最后一位加一。
3. CIDR 范围要按块边界算，题面地址不一定是规范网络号。
4. IP 分片每片都带 IP 首部，偏移按 8 字节计。
5. IPv4 分片重组在目的主机，不在中间路由器。
6. OSPF 是链路状态，RIP 是距离向量，BGP 是路径向量。
7. BGP 是 AS 之间，不是 AS 内部。
