# 应用层与网络安全

## 一、DNS

DNS 用于域名到 IP 地址的解析。历年反复考递归查询和迭代查询。

递归查询：客户端向本地域名服务器发出请求；如果本地域名服务器不能直接回答，它会代替客户端继续向其他 DNS 服务器查询，直到得到结果并返回给客户端。客户端只等最终结果。

迭代查询：服务器若不能回答，会返回下一步应查询的 DNS 服务器地址；查询方再去问下一个服务器。

真题描述“客户端向本地域名服务器发送查询，本地服务器继续向其他服务器查询，最后返回客户端”，答案是递归解析。

常见 DNS 端口：UDP 53 为主，区域传送等场景可用 TCP 53。

## 二、HTTP、FTP、邮件协议

HTTP：Web 应用协议，通常基于 TCP，默认端口 80；HTTPS 默认端口 443。HTTP 本身不负责丢失报文恢复，可靠性依赖 TCP。

FTP：文件传输协议，基于 TCP，使用两个 TCP 连接：控制连接和数据连接。控制连接默认端口 21。

SMTP：用于发送电子邮件。

POP3：用于从邮件服务器取回邮件，常见端口 110。

IMAP：用于邮件访问和同步，适合多设备管理邮箱。

必须会区分：SMTP 是发邮件；POP3/IMAP 是收/取/管理邮件；FTP 是文件传输，不是邮件发送。

## 三、端口与应用

常见端口：

| 协议 | 端口 | 下层协议 | 用途 |
|---|---:|---|---|
| HTTP | 80 | TCP | Web |
| HTTPS | 443 | TCP | 加密 Web |
| FTP 控制 | 21 | TCP | 文件传输控制连接 |
| DNS | 53 | UDP/TCP | 域名解析 |
| SMTP | 25 | TCP | 邮件发送 |
| POP3 | 110 | TCP | 邮件接收 |

## 四、网络安全核心概念

### 1. 加密

加密用于保密性，防止窃听者读懂内容。防止网络嗅探、信息被窃听，首选“加密传输”。

### 2. 数字签名

数字签名用于认证、完整性和不可否认。发送方用自己的私钥签名，接收方用发送方公钥验证。它可以验证报文是否被篡改、是否确实来自发送方。

### 3. 认证

认证用于确认身份是否合法。身份认证不等于内容保密，也不等于报文完整性校验。

### 4. 防火墙

防火墙根据规则过滤流量，规则可基于源/目的 IP、端口号、协议等。它可以控制访问，但不能防止所有攻击，也不能让已被嗅探到的明文自动变安全。

### 5. VPN

VPN 是虚拟专用网，通过公共网络建立安全的逻辑专用通道，常结合隧道、加密和认证，用于远程用户或分支机构安全接入内部网络。

### 6. NAT

NAT 是网络地址转换，解决私有地址访问公网和公网地址不足问题。NAT 不是加密技术，也不是数字签名。

## 五、典型选择题判断

防止嗅探：选加密传输。

验证报文是否被篡改和伪造：选数字签名。

控制进出网络访问：选防火墙。

连接多个城市的部门并通过 Internet 建立安全内部通信：选 VPN。

解决 IP 地址不足、内网共享公网地址：选 NAT。

发送方既要认证又要保密：先用发送方私钥签名，再用接收方公钥加密。

## 六、常见失分点

1. 把加密和数字签名混淆。加密主要保密，数字签名主要认证和完整性。
2. 防火墙不能防止所有攻击，尤其不能解决明文被嗅探后的内容泄露。
3. NAT 不提供加密隧道。
4. HTTP 丢包恢复依赖 TCP，不是浏览器和服务器应用层自己恢复。
5. SMTP 是发邮件，不是 POP3/IMAP。
